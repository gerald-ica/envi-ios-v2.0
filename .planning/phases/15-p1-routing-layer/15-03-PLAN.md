<objective>
Wire incoming deep-link URLs through `AppRouter` so future Universal Links, push-notification taps, and widget intents can all reach the same destinations as in-app taps.

Purpose: The `enviapp://` URL scheme is already registered (Phase 6 used it for OAuth callbacks). Today those callbacks are handled directly in the OAuth module. This plan adds a thin router-aware URL handler that: (a) leaves OAuth callbacks untouched, (b) parses new-style `enviapp://destination/{name}` URLs into `AppDestination` cases, (c) dispatches via `AppRouter.present(...)`. This is what unlocks future "jump to Campaign X" push-notification handling without needing another routing refactor.

Output: `ENVI/Navigation/DeepLinkRouter.swift` parses URL → AppDestination. `AppCoordinator` (or `SceneDelegate`) intercepts `application(_:open:options:)` / `scene(_:openURLContexts:)`, hands URLs to the parser, falls back to existing OAuth handling if the URL doesn't match a known destination. Unit tests cover the mapping table. No runtime changes to the current OAuth path.
</objective>

**Constraining decisions (from STATE.md + v1.1):**
- URL scheme `enviapp://oauth-callback/{provider}` is reserved for OAuth — DO NOT change its handling path or it breaks all 6 connectors
- Universal Links are deferred to v1.2+ (not necessarily this phase) — this plan prepares the seam but doesn't ship the Apple App Site Association config
- Router is now threaded through MainTabBarController (Plan 15-02) so the SceneDelegate or AppCoordinator can reach it via the root view controller

**URL grammar to support (this phase):**
- `enviapp://oauth-callback/{provider}` — unchanged, routed to existing OAuth handler
- `enviapp://destination/{caseName}` — new, parsed to `AppDestination.{caseName}` (for cases with no payload)
- `enviapp://destination/{caseName}?id={value}` — new, parsed to `AppDestination.{caseName}(id: value)` (for cases with a single ID payload)

  <name>Task 1: DeepLinkRouter.swift — parse URL → AppDestination</name>
  <files>ENVI/Navigation/DeepLinkRouter.swift</files>
  <action>Create a pure-Swift struct `DeepLinkRouter` with a single static method:
  
  ```swift
  static func destination(from url: URL) -> AppDestination?
  ```
  
  Rules:
  1. If `url.scheme != "enviapp"`, return nil.
  2. If `url.host == "oauth-callback"`, return nil (leave OAuth handling to existing code — this function is ONLY for non-OAuth destinations).
  3. If `url.host == "destination"`, parse `url.pathComponents` for the case name (first non-slash component). Parse `URLComponents.queryItems` for optional `id` / other payload params.
  4. Map the parsed case name to an `AppDestination` case. Use a central `static let caseRegistry: [String: (URL) -> AppDestination?]` so adding new destinations is a one-line change rather than a switch sprawl.
  5. On unknown case name, return nil (do NOT crash, do NOT silently route to a default).
  
  Register mappings for the existing cases that have no payload plus 2-3 that take `id:` payload so the test suite can exercise both shapes.
  
  Avoid: throwing — return nil on parse failure, let the caller decide what to do. Avoid: logging via `print` — use the existing TelemetryManager if URL parsing fails, so we have observability on bad deep links.</action>

  <done>DeepLinkRouter.swift exists, compiles, registry includes a representative sample of cases.</done>
