<objective>
Define the `AppDestination` enum and the `AppRouter` `ObservableObject` that will consolidate modal presentation across the ENVI iOS app.

Purpose: Today `NavigationCoordinator.swift` contains only Coordinator/ParentCoordinator protocols — there is no central route registry. 14+ `.sheet(isPresented: $boolState)` call sites are scattered across the 3 live tabs (HomeFeed, ChatExplore, Profile) and 14 Modal feature groups are architectural orphans because nothing presents them. This plan introduces the missing abstraction — one enum that enumerates every reachable destination + one `ObservableObject` that drives presentation — so Phase 16 can wire Agency, Commerce, Campaigns, AIFeatures, etc. from a single place.

Output: Two new Swift files — `ENVI/Navigation/AppDestination.swift` (enum) and `ENVI/Navigation/AppRouter.swift` (router). Router is `@MainActor`, conforms to `ObservableObject`, publishes a `current: AppDestination?` binding for sheet/fullScreenCover, and exposes `present(_:)`, `dismiss()`, `replace(_:)`. Covered by unit tests in `ENVITests/AppRouterTests.swift`. No existing call sites are touched yet — that is Plan 15-02.
</objective>

**Constraining decisions (from STATE.md + v1.1):**
- Hybrid UIKit/SwiftUI: top-level coordinators are UIKit (AppCoordinator, MainTabBarController, OnboardingCoordinator); per-tab content is SwiftUI. Router lives in the SwiftUI layer — do NOT try to replace MainTabBarController's UIKit nature.
- Tab count is currently 3 (ForYouGallery, ChatExplore, Profile). Phase 16 will add a 4th (Publishing). Router must support `tab(Int)` selection so Phase 16's Publishing entry points can cross-tab.
- Deep-link scheme `enviapp://` is reserved for OAuth callbacks (Phase 6). Plan 15-03 wires the router to handle additional deep-link paths without breaking OAuth.

  - `func present(_ destination: AppDestination, preferring: AppDestination.Presentation? = nil)` — routes to sheet/fullScreen/push/tab based on preferring ?? destination.defaultPresentation. If a sheet is already up and the new one is also `.sheet`, dismiss-then-present via a single-frame delay so SwiftUI's presentation machinery doesn't deadlock (known SwiftUI quirk — use `Task { @MainActor in … }` pattern).
  - `func dismiss()` — clears sheet/fullScreen whichever is non-nil.
  - `func replace(_ destination: AppDestination)` — dismiss-then-present shorthand.
  - `func selectTab(_ index: Int)` — sets selectedTab, clears sheet/fullScreen.
  
  Provide a static shared instance as a convenience, but also make it injectable — tab containers should `@StateObject var router = AppRouter()` so each tab can have isolated routing state if needed, with a `@EnvironmentObject` shared bridge for cross-tab navigation. Document this pattern in the file's top comment.
  
  Do NOT wire this into any existing views in this task — that's 15-02. This file should compile and be testable in isolation.</action>
