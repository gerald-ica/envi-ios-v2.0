<objective>
Wire the `AppRouter` into the 3 live tabs and migrate 2-4 existing `.sheet(isPresented: $boolState)` call sites to the router as a proof-of-pattern.

Purpose: Plan 15-01 built the router in isolation. This plan integrates it: the UIKit `MainTabBarController` threads an `AppRouter` instance into each `UIHostingController`'s root view via `.environmentObject`; each SwiftUI tab root attaches `.sheet(item: $router.sheet)` + `.fullScreenCover(item: $router.fullScreen)`. To keep scope tight we migrate a small number of existing sheet call sites as proof the pattern works — the rest can migrate opportunistically over time.

Output: `MainTabBarController` instantiates and owns a shared `AppRouter`. Three tab root views attach the router as an environment object and wire `.sheet(item:)` / `.fullScreenCover(item:)`. 2-4 existing inline `.sheet(isPresented:)` call sites converted to `router.present(...)`. No orphan modals wired yet — that is Phase 16's scope; this plan only proves the pipe works.
</objective>

**Constraining decisions:**
- Router is owned by MainTabBarController (the tab container) so it survives tab switches
- Each tab's root view receives it via `.environmentObject(router)` on the hosting controller's rootView
- `selectedTab` in AppRouter mirrors `MainTabBarController.currentIndex` — set up a two-way observer so programmatic `router.selectTab(2)` actually switches tabs

**Migration candidates (from grep in 15-planning):**
- ForYouGalleryContainerView.swift:33 — `.sheet(isPresented: $showSearch)` → `.search` destination

- ChatExploreView.swift:57 — `.sheet(isPresented: $showHistory)` → `.chatHistory` destination
- ChatExploreView.swift:60 — `.sheet(isPresented: $showSettings)` → `.contentLibrarySettings` destination

Do NOT migrate:
- LibraryView sheets (too deep, Phase 16 will route them from the new nav structure)
